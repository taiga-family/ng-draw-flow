In this part of the docs we are explaining some basic NgDrawFlow terms and definition. Three things you’ll be using a lot in NgDrawFlow are the model, nodes, connectors, and connections.

Data Model

DfDataModel is the complete graph value: a node array, a connection array, and an optional connection anchor strategy. It is bound through Angular forms, so saving, undo/redo and collaboration can all operate on one application-owned value.

Nodes

A node in NgDrawFlow renders an Angular component. This means it can display anything you desire. Each positioned node has X and Y world coordinates that refer to its top-left wrapper corner. To create a node, you need to extend from DrawFlowBaseNode .

The workspace itself is dynamic. It is derived from the rendered node bounds rather than from a fixed predefined board size, so it expands together with the graph content.

To delete a node, select it and press Delete or Backspace

Some of the things you might do with a custom node are:

  • Render form elements
  • Visualize data
  • Support multiple connectors

Connectors

A connector is used to link nodes together. Input and output handles have different default colors and can be changed using CSS variables. A custom node may use as many connector handles as needed. Connector ids must be unique across the graph; prefixing them with the node id is the recommended convention (example: node-1-input-1 ). More information can be found in the connector documentation .

Connectors are divided into two types: input connectors and output connectors.

To connect two nodes, simply drag an output connector from one node to an input connector of another node.

Connections

A connection links two nodes together. It is represented by a Bezier or SmoothStep SVG path, allowing its line thickness, interaction area, colors, arrowhead and label to be customized.

More details can be found in the connection lines documentation .

To delete a connection, select it and press the Delete or Backspace key .
